# CONTAINER 18-Month Startup Roadmap

Date: 2026-06-01
Status: Working startup roadmap

## Summary

CONTAINER should develop from an early lunar construction concept into a credible startup-grade technical program. The first priority is learning enough to make strong decisions, then converting that learning into focused models, low-cost tests, advisor feedback, and a fundable milestone package.

Default strategy: learn fast, prove the risky assumptions cheaply, and pitch only what has evidence behind it.

## Startup Positioning

CONTAINER is a regolith-ballasted lunar construction cell for building landing pads and berms. It is designed to land light, work heavy, and reuse local regolith as both temporary machine mass and permanent infrastructure.

Do not position CONTAINER as:

- A complete lunar construction solution.
- A final or validated design.
- Ready for integration.
- Cheaper than alternatives without a sourced cost model.
- Independent of power, dust, sintering, anchor, or autonomy risk.

## Phase 1: Learning Foundation and Project Control, Months 0-2

Objective: build the knowledge base and project-control system before making large technical or startup claims.

Actions:

- Follow the structured learning syllabus in `CONTAINER_Learning_Syllabus.md`.
- Use the baseline files in `../00_Current_Baseline` as the current technical truth.
- Maintain an evidence confidence system: calculated, project assumption, external source, test result, or design target.
- Replace final-design wording with startup-safe language: "consolidated baseline pending validation."
- Build the first advisor list across lunar systems, robotics, regolith/geotechnical, power/thermal, aerospace business, and startup/IP.

Deliverables:

- Personal learning syllabus.
- Startup truth system.
- Top 10 technical unknowns.
- Clean founder brief.
- Advisor outreach list and email templates.

Success criteria:

- Every important number has an owner, confidence level, and next verification method.
- The project can be explained honestly in less than 2 minutes.
- No public-facing material overstates maturity.

## Phase 2: Technical Credibility Sprint, Months 2-5

Objective: make the concept hard to dismiss by improving the model and focusing on the assumptions most likely to break the project.

Actions:

- Expand the existing model into a startup diligence model covering mass, power, schedule, stability, cost, and sensitivity.
- Add kill-case scenarios for sintering energy, intake throughput, anchor capacity, dust durability, battery survival, and cost.
- Convert model results into charts suitable for advisor review.
- Update the proof-versus-assumption table after each model revision.

Deliverables:

- Phase 2 model update.
- Sensitivity charts.
- Risk-ranked experiment roadmap.
- Revised technical brief.
- Current proof versus assumption table.

Success criteria:

- The model identifies the top feasibility drivers.
- Weak assumptions are visible instead of hidden.
- Advisor questions can be answered with current evidence or marked as open.

## Phase 3: Low-Cost Proof Experiments, Months 5-10

Objective: run subscale experiments before attempting an integrated machine.

Actions:

- Test regolith/simulant intake and transfer.
- Test ballast fill and dump behavior.
- Test helical anchor torque and pull-out.
- Test small gantry accuracy under load.
- Test sintering coupon or tile-joint behavior.
- Test dust exposure effects on moving parts and exposed surfaces.

Deliverables:

- Test reports with photos and videos.
- Measured values for the assumptions register.
- Prototype failure log.
- Revised architecture decision: continue, pivot, narrow scope, or retire a subsystem approach.

Success criteria:

- Each test produces measured values, not only visual proof.
- Failed tests update the design.
- Public claims are revised to match evidence.

## Phase 4: Startup Formation Package, Months 8-12

Objective: turn the technical work into a package suitable for advisors, competitions, incubators, grants, space accelerators, and early investor conversations.

Actions:

- Define the first beachhead product as repeatable lunar landing pad and berm construction using regolith ballast.
- Prepare the pitch deck, one-page investor/advisor memo, technical appendix, and demo budget.
- Start a basic IP and prior-art review.
- Document which information can be shared publicly and which should remain confidential.

Deliverables:

- 10-12 slide pitch deck.
- One-page investor/advisor memo.
- Technical appendix.
- IP/prior-art notes.
- Budget for next demonstrator.
- Advisor feedback summary.

Success criteria:

- The startup story is specific, technical, and evidence-first.
- The next funding milestone has a clear cost, output, and reason.
- The project can survive basic diligence questions without overclaiming.

## Phase 5: Integrated Demonstrator and Funding Path, Months 12-18

Objective: build a visible integrated demonstrator only after individual subsystem tests produce useful data.

Actions:

- Build a bench or yard-scale demonstrator showing fill, ballast stabilization, anchoring, gantry operation, construction analog, dump, and relocate logic.
- Measure cycle time, power draw, fill/dump behavior, anchor performance, and gantry repeatability.
- Use the demo to pursue grants, accelerator entry, a university lab partnership, or early seed conversations.

Deliverables:

- Integrated demo video.
- Measured cycle data.
- Updated cost and mass model.
- Seed/grant proposal package.
- 24-month technical development plan.

Success criteria:

- The demo shows a full operating cycle at subscale.
- The measured data improves the model.
- The next raise or partnership ask is tied to a specific technical milestone.

## Monthly Cadence

Each month should end with:

- One updated learning note.
- One updated risk or assumption entry.
- One advisor question or outreach action.
- One model, test, CAD, or document artifact.
- One decision about what to stop, continue, or narrow.
